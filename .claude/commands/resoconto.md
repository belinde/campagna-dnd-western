# Resoconto post-sessione

Prima di qualsiasi azione, leggere con `Read` il file `.claude/skills/campagna-resoconto/SKILL.md` e applicare **integralmente** quel flusso, fase per fase, senza saltare le conferme richieste al DM.

Non produrre tutto il materiale in un'unica passata. Le convenzioni permanenti del repository restano in `CLAUDE.md` (contesto permanente). Se esiste `sessione/trascrizione.md` verificato (es. dopo `/trascrizione-vc` o `/trascrizione-discord`), usarlo come **fonte primaria** degli eventi; il DM integra solo i gap non deducibili dal testo.
